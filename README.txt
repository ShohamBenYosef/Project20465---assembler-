הסבר כללי על מבנה הקבצים:

parse - ריצה ראשונה של האלגוריתם, פענוח שורות. בנוסף מחזיק מבנה parser_data ששומר את נתוני הקובץ.
commands - פענוח פקודות וארגומנטים. נקראת ע"י parse.
error - קובץ שגיאות, מכיל שגיאות נפוצות ושיטות עזר.
file - פתיחת קבצים, קריאת שורות ומילים מקובץ.
list - רשימות מקושרות, בנית צמתים, הוספה לרשימה, מחיקה וכו. 
translate - המעבר השני של האלגוריתם, כולל שיטות של כתיבה לקבצי פלטץ
utils - שיטות עזר וקבועים. 



הסבר כללי על הפרויקט:

התוכנית מקבלת שמות קבצים בארגומנט ובודקת באמצעות השיטות שבקובץ file.c האם קיים קובץ בעל שם זהה כך שמסתיים בas.
לאחר שפותחים את הקובץ, נפענח אותו שורה אחר שורה באמצעות השיטה parse הנמצאת בקובץ עם אותו השם, השיטה קוראת לפונקציות עזר ע"מ 
לבנות רשימות מקושרות (Line ו Lebel) שבעזרתן נשמור על הנתונים שנאסוף ( כמו תויות וקודי מכונה).
בנוסף, נעזר בתת-שיטות (הנמצאות בparse וב commands)  ע"מ לאפיין את השורה ולחלץ ממנה את המידע המבוקש.
לאחר ריצה על הקובץ, נעבור באמצעות השיטה parse2 (נמצאת בקובץ translate.c ) על הרשימות ששמרנו ונמלא את הערכים החסרים( של קודי המכונה וכתובות)
ונדפיס את הנתונים לקבצים המתאימים.
